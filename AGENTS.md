# pi-mixlayer

A pi provider extension that registers [Mixlayer](https://mixlayer.ai) as a model provider using Mixlayer's OpenAI-compatible Chat Completions API.

## What it does

- Fetches the available model list from `https://models.mixlayer.ai/_openrouter/models`.
- Caches the model list in `~/.pi/agent/mixlayer-models-cache.json` for one hour.
- Falls back to stale cache if the network request fails.
- Registers the models under the `mixlayer` provider with `api: "openai-completions"` and `baseUrl: "https://models.mixlayer.ai/v1"`.
- On `session_start`, selects `qwen/qwen3.5-397b-a17b` (or `qwen/qwen3.6-35b-a3b` as fallback) as the default model if no model is selected yet.

## Layout

```
.
├── src/              # TypeScript source files
│   └── index.ts      # Extension entry point
├── dist/             # Compiled JavaScript output (generated by tsc)
├── package.json      # Package manifest and pi extension config
├── tsconfig.json     # TypeScript config
├── README.md         # Human-facing documentation
└── AGENTS.md         # This file
```

## Build

```bash
npm run build   # Compile src/ into dist/
npm run check   # Type-check without emitting files
npm run clean   # Remove dist/
```

The `pi.extensions` field in `package.json` points to `./dist/index.js`, so pi loads the compiled output, not the TypeScript source.

## Adding modules

Create new `.ts` files under `src/` and import them from `src/index.ts` using relative ESM paths:

```ts
import { foo } from "./foo.js";
```

TypeScript compiles all `src/**/*.ts` files into `dist/` while preserving the relative module structure.
